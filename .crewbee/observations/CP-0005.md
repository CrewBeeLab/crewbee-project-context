# CP-0005 Minimal Framework Optimization

## Summary

Applied the minimalism principle to the framework implementation.

## Key Changes

- `init` now reads scaffold files directly from `templates/crewbee-template/`, eliminating the duplicated in-code template source.
- Removed unused schema files and removed `schemas/` from package/build outputs.
- Limited the CrewBee integration bridge to compact prompt primer plus `project_context_read`, `project_context_search`, and `project_context_finalize`.
- Added tests for absent `.crewbee/` behavior and minimal CrewBee tool execution.

## Verification

- `npm run diagnostics` passed.
- `npm run typecheck` passed.
- `npm test` passed with 11 tests.
- `npm run build` passed.

## Follow-ups

- Keep the baseline stable; only add release hardening or new features with explicit acceptance criteria.
