# CP-0015 — OpenCode Desktop Project Context update observability

## Summary

Implemented official OpenCode Desktop observability for automatic Project Context update while keeping prepare on the fast system-transform path.

## Material Changes

- Automatic prepare remains local-I/O based and uses `experimental.chat.system.transform` for the compact model brief; it also writes a Desktop-visible `session.prompt(noReply)` status message with `ignored: true` so the visible prepare summary does not enter later LLM context.
- Automatic update now writes full update context to a one-time private `.crewbee/.prjctxt/cache/update-jobs/` JSON payload, then submits a compact `subtask` part to the parent session through `promptAsync`; OpenCode's built-in Task flow creates the clickable task execution card and child session linkage.
- The parent update Task prompt references only the private payload file and does not embed full latest-user-request, assistant-final-summary, git-status, git-diff, or verification-output content into the parent session.
- The direct Task guard still blocks ordinary direct calls to `project-context-maintainer`, but allows the internal `project_context_update` subtask command.
- Tests now assert prepare calls `session.prompt(noReply)` with a non-synthetic, `ignored: true` visible summary, update launches an official subtask targeting `project-context-maintainer`, the private update payload contains the full summary/verification evidence, and the payload is deleted after the internal task completes.
- README, CrewBee integration docs, implementation snapshot, state, and handoff were updated to reflect the new observability behavior.
