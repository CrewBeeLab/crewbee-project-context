# CP-0009 Minimal Attention Runtime Alignment

## Summary

Aligned the project design with the OpenCode + CrewBee minimal-attention runtime model.

## Key Changes

- Rewrote the Chinese guide around plug-and-play OpenCode/CrewBee integration.
- Defined the main tool surface as `project_context_prepare`, `project_context_search`, and `project_context_finalize_request`.
- Removed `project_context_read` from the CrewBee bridge tool surface.
- Clarified that CLI is internal/debug oriented, not the primary user workflow.
- Updated capsule wording so main agents are not given scaffold file read order.
- Added `ProjectContextMaintainer` as the internal execution boundary.
- Added lazy bootstrap on `project_context_finalize_request` when `.crewbee/` is absent.
- Reorganized source directories into `workspace/`, `capsule/`, `maintainer/`, and split CrewBee integration into extension, prompt fragment, tool definitions, tool handlers, and internal-agent metadata.
