# CP-0008 TypeScript OOP Conversion

## Summary

Converted the implementation from JavaScript modules to TypeScript and reorganized the core around small object-oriented classes.

## Key Changes

- Added TypeScript build configuration and dev dependencies.
- Updated package exports and CLI bin to use `dist/` output.
- Introduced `ProjectContextService` as the internal facade.
- Converted store, scaffold, parser, primer, search, update, finalize, and CrewBee bridge into typed classes.
- Removed `contextDir` as a public option and removed `context_dir` from config files; `.crewbee/` remains the only production context directory.
- Updated `.crewbee/IMPLEMENTATION.md` important paths to match the TypeScript source layout.

## Verification

- `npm run diagnostics` passed.
- `npm run typecheck` passed.
- `npm test` passed with 10 tests.
- `npm run build` passed.

## Follow-ups

- Keep object-oriented structure small and avoid new abstractions without real callers.
