# CP-0015 — OpenCode Desktop Project Context update observability

## Summary

Implemented official OpenCode Desktop observability for automatic Project Context update while keeping prepare on the fast system-transform path.

## Material Changes

- Automatic prepare remains a local-I/O `experimental.chat.system.transform` injection and does not create no-reply parent-session messages.
- Automatic update now submits a `subtask` part to the parent session through `promptAsync`; OpenCode's built-in Task flow creates the clickable task execution card and child session linkage.
- The direct Task guard still blocks ordinary direct calls to `project-context-maintainer`, but allows the internal `project_context_update` subtask command.
- Tests now assert prepare does not call `session.prompt(noReply)` and update launches an official subtask targeting `project-context-maintainer`.
- README, CrewBee integration docs, implementation snapshot, state, and handoff were updated to reflect the new observability behavior.

## Verification
